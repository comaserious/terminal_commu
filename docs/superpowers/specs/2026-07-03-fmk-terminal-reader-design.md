# FMK 해외축구 터미널 리더 설계

## 개요

`fmk` 명령 하나로 FM코리아 해외축구 게시판의 공개 글 목록, 본문, 댓글을 터미널 안에서 읽는 개인용 macOS 애플리케이션을 만든다. 애플리케이션은 읽기 전용이며 로그인, 글쓰기, 추천, 이미지 표시는 지원하지 않는다.

대상 게시판은 `https://www.fmkorea.com/football_world` 한 곳으로 고정한다. 구현은 Python과 Textual을 사용한다.

## 실행 환경

- Conda 환경: `basic-env`
- 확인된 Python 버전: 3.12.13
- 개발, 테스트, 실행 전에 `conda activate basic-env`를 사용한다.
- 프로젝트 의존성은 `pyproject.toml`에 선언하고 활성화된 `basic-env`에 설치한다.

## 목표

- `fmk` 실행 즉시 해외축구 게시판 목록을 보여준다.
- 넓은 터미널에서는 왼쪽에 목록, 오른쪽에 선택한 글의 본문과 댓글을 표시한다.
- 방향키 중심으로 탐색하며 마우스 없이 모든 읽기 기능을 사용할 수 있다.
- 사이트 요청을 최소화하고 캐시를 사용해 `429 Too Many Requests` 위험을 줄인다.
- FM코리아 HTML 구조 변경을 UI 코드와 분리된 파서에서 대응할 수 있게 한다.

## 제외 범위

- 로그인, 글쓰기, 댓글 작성, 추천, 비추천, 스크랩
- 이미지와 동영상의 다운로드 또는 터미널 렌더링
- 해외축구 이외의 게시판과 임의 URL 지원
- CAPTCHA, 로그인 요구, 접근 차단 우회
- 다른 사용자 배포를 위한 설치 패키지와 다중 운영체제 지원

## 사용자 경험

### 실행

사용자는 터미널에서 `fmk`를 실행한다. 별도 URL 인자는 받지 않는다. 시작 시 유효한 캐시가 있으면 목록을 즉시 표시하고, 캐시가 없을 때만 네트워크에서 목록을 가져온다.

### 화면 구성

- 상단: 앱 이름, 게시판 이름, 마지막 갱신 시각, 캐시 또는 네트워크 상태
- 왼쪽 38%: 글 제목, 카테고리, 추천 수, 댓글 수, 작성 시각을 포함한 목록
- 오른쪽 62%: 제목과 메타데이터, 본문 텍스트, 댓글 목록
- 하단: 현재 사용할 수 있는 키 도움말

터미널 폭이 100열 미만이면 단일 화면 모드로 전환한다. 목록에서 글을 열면 본문 화면으로 이동하고 `Esc`로 목록에 돌아간다.

### 키보드 조작

- `↑` / `↓`: 목록 선택 또는 현재 본문·댓글 영역 한 줄 스크롤
- `←` / `→`: 게시판 페이지 또는 댓글 페이지 이동
- `Enter`: 선택한 글 열기
- `Tab`: 목록과 본문·댓글 영역 사이 포커스 전환
- `Esc`: 좁은 화면의 본문에서 목록으로 돌아가기
- `r`: 현재 목록 또는 글을 명시적으로 새로고침
- `q`: 종료

## 아키텍처

애플리케이션을 UI, 서비스, 사이트 클라이언트, 캐시의 네 계층으로 분리한다.

### Textual TUI

화면 배치, 포커스, 키 입력, 로딩 상태와 오류 메시지를 담당한다. HTML 선택자나 HTTP 응답 형식을 알지 못하고 서비스 계층의 데이터 모델만 사용한다.

### Board Service

목록과 글 로딩을 조정하고 현재 페이지와 선택 상태를 관리한다. 캐시 우선 조회, 지연 로딩, 요청 속도 제한, 오류를 사용자 메시지로 변환하는 책임을 가진다.

### FMK Client

`httpx`로 공개 페이지를 읽고 `BeautifulSoup`으로 HTML을 파싱한다. 목록, 게시글, 댓글을 내부 데이터 모델로 정규화한다. 이미지와 동영상 요소는 다운로드하지 않고 각각 `[이미지 생략]`, `[동영상 생략]` 텍스트로 치환한다.

파서는 최소한 다음 URL 형식을 처리한다.

- 게시판 목록: `https://www.fmkorea.com/football_world`
- 게시글: `https://www.fmkorea.com/{document_srl}` 또는 동등한 `index.php` 형식
- 댓글 페이지: `document_srl`, `mid=football_world`, `cpage`를 포함한 형식

### 로컬 캐시

Python 표준 라이브러리 `sqlite3`로 `~/.cache/fmk-reader/cache.db`를 관리한다. URL, 응답 시각, 파싱된 JSON 데이터를 저장한다.

- 목록 TTL: 1분
- 본문 TTL: 30분
- 댓글 TTL: 2분

캐시는 성능 최적화뿐 아니라 요청 실패 시 마지막 성공 데이터를 제공하는 용도로 사용한다.

## 데이터 모델

- `PostSummary`: 글 ID, 제목, 카테고리, 작성자, 작성 시각, 조회 수, 추천 수, 댓글 수, URL
- `PostDetail`: `PostSummary` 정보, 정리된 본문 블록, 외부 링크, 전체 댓글 수
- `Comment`: 댓글 ID, 작성자, 내용, 작성 시각, 답글 깊이
- `PageResult[T]`: 항목 목록, 현재 페이지, 다음·이전 페이지 존재 여부, 데이터 출처와 갱신 시각

모델은 UI와 파서 사이의 계약이다. 파서의 CSS 선택자가 바뀌어도 모델 형식은 유지한다.

## 네트워크 정책

- 네트워크 요청은 동시에 하나만 실행한다.
- 요청 시작 사이에 최소 2초 간격을 둔다.
- 연결과 읽기에는 유한한 타임아웃을 설정한다.
- 서버의 `Retry-After`가 있으면 이를 존중한다.
- `429` 응답에는 자동 반복 요청을 하지 않고 캐시를 사용한다.
- `r`을 눌러도 직전 요청 후 2초가 지나지 않았으면 즉시 요청하지 않는다.
- 로그인, CAPTCHA, 차단 페이지가 감지되면 중단하고 우회를 시도하지 않는다.

## 오류 처리

- 네트워크 타임아웃: 캐시가 있으면 캐시 표시와 함께 상태 바에 경고를 표시한다.
- `429`: 재시도를 중단하고 사용 가능한 캐시와 마지막 갱신 시각을 표시한다.
- `403` 또는 차단 페이지: 읽기 기능을 중단하고 브라우저에서 직접 확인하라는 메시지를 표시한다.
- 파싱 실패: 빈 화면으로 성공 처리하지 않고 `사이트 구조가 변경되었을 수 있음` 오류를 표시한다.
- 일부 필드 누락: 제목이나 본문처럼 필수 필드는 오류로 처리하고, 조회 수 등 선택 필드는 기본값으로 표시한다.
- 캐시 손상: 해당 레코드를 무시하며 앱 전체를 종료하지 않는다.

## 테스트 전략

### 단위 테스트

- 저장된 목록 HTML에서 일반 글, 공지, 카테고리, 통계와 페이지 정보를 파싱한다.
- 저장된 게시글 HTML에서 제목, 메타데이터, 본문, 링크, 미디어 대체 문구를 파싱한다.
- 저장된 댓글 HTML에서 일반 댓글과 답글 깊이, 다음 댓글 페이지를 파싱한다.
- 누락되거나 변경된 필수 선택자에 명확한 파싱 오류가 발생한다.
- TTL, 캐시 만료, 손상 레코드 무시, `429` 캐시 폴백을 검증한다.

### UI 테스트

- 목록 선택, 글 열기, 포커스 전환, 방향키 스크롤과 종료 키를 Textual 테스트 파일럿으로 검증한다.
- 100열 전후에서 분할 화면과 단일 화면 전환을 검증한다.
- 로딩, 캐시, 네트워크 오류 상태가 화면에 표시되는지 검증한다.

### 수동 스모크 테스트

실제 사이트 요청은 최소 횟수로 수행한다. 목록 한 번, 게시글 한 번, 댓글 첫 페이지 한 번을 읽어 모델과 화면 표시를 확인한다. 자동 테스트에서는 실제 사이트에 요청하지 않는다.

## 프로젝트 구조

```text
terminal_community/
├── pyproject.toml
├── README.md
├── src/fmk_reader/
│   ├── app.py
│   ├── models.py
│   ├── service.py
│   ├── cache.py
│   ├── client.py
│   ├── parser.py
│   └── styles.tcss
└── tests/
    ├── fixtures/
    ├── test_parser.py
    ├── test_cache.py
    ├── test_service.py
    └── test_app.py
```

각 모듈은 한 가지 책임만 가진다. `app.py`는 화면, `service.py`는 흐름, `client.py`는 HTTP, `parser.py`는 HTML 해석, `cache.py`는 SQLite 저장을 담당한다.

## 완료 기준

- `conda activate basic-env` 후 의존성을 설치하고 테스트와 앱을 실행할 수 있다.
- macOS 터미널에서 `fmk` 명령으로 앱이 시작된다.
- 해외축구 게시판 목록을 읽고 방향키로 선택할 수 있다.
- 선택한 글의 본문과 댓글을 이미지 없이 터미널에서 읽을 수 있다.
- 넓은 화면의 분할 UI와 좁은 화면의 단일 UI가 모두 동작한다.
- 캐시 TTL과 요청 간격이 적용되고 `429`, 타임아웃, 파싱 오류가 명확히 표시된다.
- 단위 테스트와 Textual UI 테스트가 실제 사이트 요청 없이 통과한다.
- 최소 수동 스모크 테스트로 실제 공개 페이지 읽기를 확인한다.
